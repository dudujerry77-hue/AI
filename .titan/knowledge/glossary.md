# Glossary

- **Titan AI** — the autonomous software engineering operating system this governance layer serves; also refers to the orchestrating agent role when multiple AI models coordinate.
- **`.titan/`** — the permanent governance/memory directory at the repository root, described by `constitution.md`.
- **Phase** — a discrete, sequential unit of work defined in `roadmap.md` and detailed in `phases/`.
- **ADR (Architecture Decision Record)** — a recorded, binding decision in `decisions.md`, including context, alternatives, and consequences.
- **Session** — one continuous unit of work by one agent (or human), logged in `sessions/`.
- **Agent** — any AI model (Claude, Codex, Lovable, Gemini, Titan AI) or human acting on the repository under this governance system.
- **Definition of Done** — the seven-point completion bar defined in `constitution.md` Section 5.
- **Fabrication** — presenting fake/mocked/hardcoded results or unverified claims as real; explicitly prohibited throughout this governance system.
- **Governance hierarchy** — the precedence order of `.titan/` documents defined in `constitution.md` Section 2 and `rules/00-supremacy.md`.
