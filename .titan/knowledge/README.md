# knowledge/

Durable reference knowledge that doesn't fit neatly into the process documents above — glossary, FAQ, and any domain knowledge future agents need but that isn't itself a governance rule.

## Contents

- `glossary.md` — shared vocabulary so every agent means the same thing by the same term.
- `faq.md` — recurring questions and their settled answers, to avoid re-litigating the same question every session.

## Rule

If you find yourself explaining the same background concept to a new agent/session more than once, it belongs here, not just in a session log.
