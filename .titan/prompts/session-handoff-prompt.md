# Prompt: Session Handoff / Cold Start

Use this when a brand-new agent (any model) with zero conversation history needs to resume work.

---

You have no prior memory of this project. Everything you need is in the `.titan/` directory at the repository root. Do the following, in order, before writing any code or making any claims about project status:

1. Read `.titan/constitution.md` fully — it is the supreme governance document.
2. Read `.titan/master_plan.md` and `.titan/roadmap.md` for overall direction.
3. Read `.titan/current_phase.md` and `.titan/project_state.json` for exactly where things stand right now.
4. Read the most recent 3 files (by date) in `.titan/sessions/`.
5. Skim `.titan/decisions.md` for binding decisions, especially any near the current phase.
6. Read `.titan/architecture.md`, `.titan/tech_stack.md`, `.titan/coding_standards.md`, `.titan/naming_conventions.md`, `.titan/security_policy.md`, `.titan/testing_strategy.md`, `.titan/deployment_strategy.md` before making any technical decision that touches those areas.

Once done, state back: the current phase, what was last completed, what is next, and any open risks or blockers you found — before proceeding with new work.
